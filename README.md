# 영주시 버스 노선 시각화 분석

> 공공데이터 기반 버스 정류장 데이터를 SQL로 관리하고 R로 시각화한 프로젝트

---

## 프로젝트 배경

경상북도 영주시 정책 공모전에서 분산된 관광 자원을 하나로 연결하는 
새로운 관광 노선을 제안하기 위해 진행한 분석 프로젝트입니다.
시장과 캠핑장을 지나는 2, 30, 330, 630번 노선을 대상으로 
데이터 수집, 정제, 시각화를 담당했습니다.

---

## 기술 스택

| 분류 | 기술 |
|------|------|
| 데이터 수집 | 공공데이터포털 |
| DB 저장 및 조회 | MySQL, DBeaver |
| 시각화 | R, leaflet |
| DB 연결 | RMySQL, DBI |

---

## 데이터 정합성 문제 및 해결

공공데이터를 수집하는 과정에서 아래 3가지 정합성 문제가 발생했습니다.

- 연도별로 추가/삭제된 정류장 존재
- 상행/하행 미구분
- 정류장 코드가 6자리 또는 8자리로 혼재

쿼리로 일괄 처리하기보다 카카오/구글 지도를 직접 확인하며
변경사항을 조사하고 수동으로 수정했습니다.
정제 이후 노선별 정류장 데이터를 즉시 조회하고
위도·경도 테이블과 결합하여 시각화에 활용할 수 있었습니다.

## 프로젝트 구조
```
yeongju-bus-analysis/
├── data/
│   ├── yeongju_bus.csv       # 영주시 버스 정류장 현황 (공공데이터포털)
│   ├── num2.xlsx             # 2번 노선 정류장 코드
│   ├── num30.xlsx            # 30번 노선 정류장 코드
│   ├── num330.xlsx           # 330번 노선 정류장 코드
│   └── num630.xlsx           # 630번 노선 정류장 코드
├── r/
│   └── visualization.R       # leaflet 시각화 코드
└── output/
    └── map.png               # 지도 결과물 (추가 예정)
```

---

## 실행 방법

1. MySQL에 `yeongju_bus.csv` 및 노선별 xlsx 데이터 적재
2. `r/visualization.R` 실행
3. RStudio Viewer에서 지도 확인
